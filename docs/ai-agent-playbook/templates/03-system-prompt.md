# System Prompt Template

> Phase 3 output. Versioned in source control with a changelog.

Each section below is a distinct concern — keep them separated. Onboard the model like a new colleague: tell it what to do (not just what to avoid), and include the *why* behind key instructions.

---

## Role

What the agent is and what it is for. One or two sentences.

## Guidelines

How the agent should approach its work — reasoning style, tone, level of caution. Stated positively.

## Policies

Hard rules the agent must follow. State the trade-off both ways when relevant (cost of acting *and* cost of not acting).

## Tool usage

When to call which tool. When *not* to call a tool. How to interpret results.

## Output format

The expected output contract. Schema if structured. Be unambiguous.

## Edge cases

What to do on: incomplete input, ambiguous requests, out-of-scope queries, repeated failure to understand.

## Examples

3–5 diverse, representative examples. For reasoning-heavy tasks, include the worked reasoning, not just the final answer.
