## How to use this

- A category is added when ~the same failure shows up across multiple annotated traces.
- *Frequency* is recomputed each review cycle — it ranks what hurts most by volume, not by what is memorable.
- *Open improvement* points to the change being worked on (prompt edit, tool fix, retrieval tweak, etc.).
- Categories die when fixed, merge when redundant, split when too coarse.
